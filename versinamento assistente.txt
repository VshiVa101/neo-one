## 1. Project Goal
Neo-One Art Hub is a custom uncensored art website for the artist Neo-One.

The project must work as:
- a real artistic website
- a premium immersive experience
- a practical portfolio project proving Leo can design, direct and ship a custom digital product

The site must feel:
- premium
- underground
- brutal / provocative
- cyber / dystopian
- custom-made
- not template-like
- visually immersive but still buildable with low budget

---

## 2. Strategic Constraints
- budget near zero
- no enterprise tools
- no heavy backend complexity if avoidable
- avoid overengineering
- low-cost / no-code / open-source preferred where sensible
- Leo is not a backend developer
- project must ship fast
- priority is execution, not theoretical purity
- avoid solutions that create long-term maintenance burden

SQLite has been abandoned. The project uses Neon Postgres from the start to avoid double work (local data that doesn't transfer to production).

Neon free tier limitation: database sleeps after 5 min of inactivity. First visitor after sleep waits ~2-3 seconds extra. Acceptable for MVP.

Cloudinary free tier: 25 credits/month. With ~1000 visits/month and images at ~300KB, estimated usage is ~7 credits/month. Enormous margin.

Image upload flow: Neo uploads images via Payload admin (`/admin`). Payload sends them to Cloudinary. The site reads them from Cloudinary. Neo never touches Cloudinary directly.

Domain: Neo already owns it. Will be connected to Vercel at deploy.

During building: Next.js + Payload run on Leo's Mac. Database is already on Neon (cloud). Images already on Cloudinary (cloud). Data is the same locally and in production.

After deploy: Next.js + Payload run on Vercel instead of Leo's Mac. Database and images stay where they are. Nothing moves. Neo accesses admin at `www.hissite.com/admin`.

## 20. Working Method
Agreed workflow:

### Three-layer versioning:
1. **Git** — absolute source of truth for code. Physical checkpoints with hash.
2. **Local agent** (Gemini / Opencode / Antigravity) — executes code changes, inspects files, reports. Has its own persistent context file (`PROJECT_CONTEXT_NEO_ONE_MASTER.md`) in the project.
3. **ChatGPT** — strategic brain. Holds full project context, decides priorities, writes prompts for agent, critiques outputs. Versioned via this Master Context file which Leo stores and restores when needed.

### ChatGPT should:
- help decide priority
- keep scope controlled
- write precise prompts for local agents
- critique outputs honestly
- not blindly trust "done" messages
- explain in natural language when Leo is confused

### Local agents should:
- inspect local code
- make specific edits
- report actual files changed
- verify real behavior when possible

### Leo should:
- test visually
- send screenshots
- challenge false positives
- keep final taste decisions
- ask for plain language whenever needed

27. Implementation Notes
Cart / Form Preview Logic
The inquiry form shows previews of selected items (artworks and/or events).
These previews use the SAME images already in the DB (mainImage for artworks, previewImage for signals).
No separate upload needed. Frontend renders them smaller via CSS.
This keeps the admin experience simple for Neo.
Availability Select Values
The availability field uses exactly these values across the whole site:

comprabile — can be purchased directly
ordinabile — can be ordered / commissioned
non_disponibile — not available
Slug Strategy
All sluggable entities (clusters, categories, artworks, signals) have a slug field.
For MVP, slugs are entered manually by Neo in admin.
Auto-generation from title can be added later if needed.

richText Usage
Only ONE field in the entire project uses richText: shippingPaymentNotice in cart-settings.
Everything else uses text or textarea to keep the admin simple.

Sort Order Convention
All sortOrder fields: lower number = appears first. Default is 0.

Image Storage
All images uploaded via Payload admin are stored on Cloudinary.
Payload uses the Cloudinary storage adapter.
Neo never interacts with Cloudinary directly.
The Media collection in Payload handles the upload flow transparently.
