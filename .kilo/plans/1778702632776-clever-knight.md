# Plan: Aumentare limite upload file audio a 10MB

## Problema
Caricamento file MP3 nel campo `sampleAudio` della collection `Artworks` (cluster Rumore) fallisce frequentemente a causa di limiti di dimensione nel flusso di upload.

## Root Cause
- `clientUploads: false` (`payload.config.ts:78`) forza i file a passare attraverso il server Next.js prima di andare su Cloudinary
- Su Vercel, le serverless functions hanno un limite di **4.5MB** per il body delle richieste
- La collection `Media.ts` non ha un `maxFileSize` esplicito
- I file audio MP3 (anche brevi) superano facilmente 4.5MB

## File da Modificare

### 1. `src/collections/Media.ts` — Aggiungere `maxFileSize: 10MB`
Aggiungere `maxFileSize: 10485760` (10MB in bytes) alla configurazione `upload` della collection Media.  
Questo dice a Payload di accettare file fino a 10MB.

### 2. `next.config.ts` — Aumentare body size limit
Aggiungere `serverActions: { bodySizeLimit: '10mb' }` per aumentare il limite di body delle richieste lato Next.js, così che anche senza `clientUploads` il server accetti file più grandi.

### 3. `src/payload.config.ts` — Attivare `clientUploads: true` (SOLUZIONE MIGLIORE)
Cambiare `clientUploads: false` → `clientUploads: true` nel plugin Cloudinary.  
Effetto: i file vanno direttamente dal browser a Cloudinary, bypassando completamente il server Next.js e il limite Vercel.  
Sicurezza: le credenziali API key/secret rimangono lato server; solo la cloud name viene esposta.

## Opzione Consigliata
Applicare **tutti e 3 i punti** per massima robustezza:
- Punto 3 risolve il problema root (limite Vercel)
- Punti 1 e 2 sono safety net per qualsiasi scenario server-side

## Ordine di Implementazione
1. `Media.ts` — `maxFileSize: 10485760`
2. `next.config.ts` — `serverActions.bodySizeLimit: '10mb'`
3. `payload.config.ts` — `clientUploads: true`
